# 文件公文化 Roadmap（持續精進版）

## 0. 產品目標與原則
- 將口語、紀錄式內容轉為正式公文語氣，提升文書品質與一致性。
- 嚴守「不虛構、不中斷脈絡、不改變核心事實」。
- 提供可追溯流程（輸入、輸出、摘要、歷程、錯誤碼）以利稽核與優化。
- 在內網環境下維持可接受延遲與穩定性，支援日常高頻使用。

## 4. 驗收指標（Definition of Done）
- [ ] 正確性：空白輸入、超長輸入、模式錯誤均回傳可判讀錯誤碼。
- [ ] 品質：抽樣檢核「不偏離原文核心事實」達成率 >= 98%。
- [ ] 效能：一般長度文本（約 1,000 字）平均處理時間 <= 8 秒（內網基準）。
- [ ] 可用性：可複製、回填再改寫、下載，流程無阻斷。
- [ ] 可追溯性：可依使用者/時間/模式/字數/結果狀態查詢歷程。

## 5. 風險與對策
- [ ] 風險：LLM 改寫過度導致事實偏移。
  - 對策：加入實體與日期比對、提供「保守改寫」開關。
- [ ] 風險：敏感資料外洩。
  - 對策：預處理遮罩 + 匯出前再掃描 + 權限分級下載。
- [ ] 風險：高峰時段延遲上升。
  - 對策：請求佇列、逾時保護、分級降載策略。
