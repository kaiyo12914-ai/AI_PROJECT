## 設定檔
`webproj\settings.py`
## 設置參考
```
PORTAL_ACL = {

    "portal": ["網頁系統管理員"],

    "student": ["PUBLIC"],

    "translator": ["ALL_AUTHENTICATED"],

    "comment": [

        "ALL_AUTHENTICATED",

        "人事資訊管理",

        "服務中心人員群組",

        "中心雲端網頁通用群組",

    ],
}
```

---
專案的權限設置主要位於以下兩個核心部分：

1. **`webproj/settings.py` 中的權限配置**：
    
    - `PORTAL_ACL_ENABLED`: 控制是否啟用權限檢查，預設為 `False`（需在 `.env` 檔案中設置為 `True` 才會生效）。
    - `PORTAL_ACL_BACKEND`: 決定權限來源，預設為 `"AUTO"`，根據 `MODEL_TYPE` 自動切換至 Oracle 或 Django 本地權限系統。
    - `PORTAL_ACL`: 定義各功能節點的存取規則（如 `doc`, `comment` 等），並指定允許的群組（如 `"ALL_AUTHENTICATED"`、`"人事資訊管理"` 等）。
2. **權限驗證邏輯**：
    
    - 在 `webapps/portal/acl.py` 中，透過 `can_access(user, node)` 函數進行權限檢查。
    - 若啟用 ACL，系統會依據使用者群組（來自 Oracle 或 Django）與規則比對，決定是否放行。

### 關鍵路徑：

- **權限規則定義**：`webproj/settings.py` 中的 `PORTAL_ACL`。
- **權限驗證邏輯**：`webapps/portal/acl.py`。
- **環境變數設置**：`.env` 檔案。

### 如何啟用權限檢查：

若要啟用權限控管，請在 `.env` 檔案中添加以下設定：

```ini
PORTAL_ACL_ENABLED=True
PORTAL_ACL_BACKEND=ORACLE  # 或 "DJANGO"
```

### 備註：

- 若 `.env` 不存在於當前工作目錄，請確認專案根目錄或相關配置路徑。
- Oracle 資料庫權限需依賴 `VIEW_ZZ_USER_GROUP_ACL` 表中的群組資訊。

若需要進一步調整權限規則，可直接修改 `webproj/settings.py` 中的 `PORTAL_ACL` 或相關環境變數。
