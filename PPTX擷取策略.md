# PPTX 擷取策略

本文記錄 `text2pptx` 的「抽取策略」與「套用策略」。兩者必須一一對應，避免抽取結果後續套用時無法回填到正確版面或文字框。
目標是「還原後視覺上盡量和原簡報一致」，後續更換文字內容即可獲得新簡報。
## 一致原則

1. 抽取時的版型名稱，必須能被套用端直接識別。
2. 套用時的版型選擇，必須優先使用抽取端輸出的版型名稱。
3. 若版型名稱無法匹配，才退回 `slide_type` 做備援。
4. 範例文字的第一行格式必須固定為 `[版型N]標題`。
5. 文字框本體、位置、大小、字形、顏色與段落格式都必須保留，只清除文字內容。

## 抽取策略

### 1. 來源

- 載入來源 `PPTX`。
- 取得每張投影片對應的 layout 與出現順序。

### 2. 版型命名

- 依檢視母片時的順序，將可用 layout 統一命名為 `版型1`、`版型2`、`版型3`...
- 抽取結果與母片預覽都使用同一套命名。

### 3. 範例抽取

- 範例內容只保留純文字。
- 每張投影片輸出格式固定為：

```text
[版型N]投影片標題
內文1
內文2
...
```

- 這個格式是給 `text2pptx` 直接貼用的，不輸出 XML 或 JSON schema。

### 4. 母片抽取

- 母片抽取時，layout 依序命名為 `版型1`、`版型2`...
- 抽取結果可保留原始 `template_schema` 作為輔助資料，但對外顯示的名稱仍以 `版型N` 為主。
- 文字框本體要保留，僅移除文字內容。

## 套用策略

### 1. 版型對應

- 套用端先讀取範例文字第一行的 `[版型N]`。
- 若找到對應的 layout 名稱，直接套用該 layout。
- 若找不到，再退回依 `slide_type` 選擇相近版型。

### 2. 文字框對應

- 套用端不得只填第一個文字框。
- 應依 layout 上實際可用的文字框順序，把文字內容分配到對應文字框。
- 若同一頁有多個文字框，應盡量依位置由上到下、由左到右對應。

### 3. 保留原則

- 保留文字框本體。
- 保留文字框位置與大小。
- 保留字形、字級、顏色與段落格式。
- 保留圖片、圖形、表格、圖表與其他非文字物件。
- 只移除投影片中的文字內容，不移除物件本體儲存文字。

## 流程

1. 載入來源 `PPTX`。
2. 抽取母片 layout，依順序命名為 `版型1、版型2...`。
3. 抽取範例文字，輸出 `[版型N]標題` 格式。
4. 套用時先以 `版型N` 對應 layout。
5. 再依文字框順序把內容填入對應文字框。
6. 若無法匹配，再退回 `slide_type` 與預設文字框邏輯。
7. 輸出 `PPTX`。

## 驗證重點

- 抽取出的 `版型N` 必須和套用時的 `版型N` 完全一致。
- 文字框填值後，母片原本的版面樣式不能被破壞。
- 若 layout 對上但文字未進框，視為套用端缺陷。
- 若抽取文字沒有帶出 `[版型N]`，視為抽取端缺陷。
